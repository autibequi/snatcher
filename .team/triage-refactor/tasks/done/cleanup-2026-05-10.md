## Retro

**O que funcionou:**
- Audit em paralelo (3 ExplorePlus) deu mapa completo em ~6 min — depois validei manualmente cada finding antes de agir.
- Split do `sql_store.go` por sed+ranges com `goimports` em seguida: zero conflito de imports, build passou primeira tentativa.
- Migration nova + field config + fallback inline: fix completo do rate-limit sem breaking change.

**O que aprendi (vai pra memória do harness):**
- Audit agents (Haiku) exageram em "duplicação clara": casos como `api.ts` vs `apiClient.ts` (semânticas diferentes), `models.NullString` ("wrapper vazio") e `GTM-XXXX` ("hardcoded") eram interpretações superficiais. **Sempre validar manualmente antes de refatorar.**
- `scraperbridge` virou exemplo de "package que parece desnecessário mas tem razão arquitetural" (quebra import cycle). Documentei explicitamente no header.
- `models.Null*` parecia boilerplate mas tem `MarshalJSON` customizado — deletar quebraria contrato JSON.

**Não escopo (deferred):**
- AppConfig sub-struct split (quebra contrato HTTP — migration + frontend update)
- Error wrapping global nas boundaries (~901 sites, baixo ROI mecânico)
- Consolidar `api.ts` + `apiClient.ts` no frontend (toca auth flow)
